# **Проект реализует API видео платформы, на которой пользователи могут публиковать свои видео**

**Функционал**:
=

* ### Регистрация

POST `/v1/register/`

---

* ### Добавление видео 
(через админку http://localhost:8000/admin)

---

* ### Получения списка с деталями опубликованных видео, а также неопубликованных, если запрашивающий - их автор 
 
GET `/v1/videos/`

---

* ### Получение деталей о видео по его ID, если оно опубликовано или запрашивается автором. Работает query-запрос ?user_expand=true

GET `/v1/videos/{id}`

GET `/v1/videos/{id}?user_expand=true`

---

* ### Постановка лайка с защитой от дубликатов, самолайков и установки на неопубликованное видео

POST `/v1/videos/{id}/likes/`

---

* Получение списка ID всех опубликованных видео

GET `/v1/videos/ids/`

---

* Получение статистики (видео-лайки) методами подзапроса и GROUP BY 

GET `/v1/videos/statistics-subquery/` и 

GET `/v1/videos/statistics-group-by/`

---

# **Админка**

Работа с видео:
Отображаемые поля: id, name, owner, is_published, total_likes, created_at
Фильтрация: по статусу (опубликовано/черновик) и по дате создания


---

# **Как запустить**

---
Перейти в корень проекта (video_platform)

`cd ./video_platform`

---

Настройка окружения:

`cp .env.example .env` - создать файл с переменными окружения

`openssl rand -base64 50` - сгенерировать секретный ключ, скопировать и вставить в .env в переменную **SECRET_KEY**

При необходимости изменить базовые секреты БД на свои

---

Запуск проекта:

`docker compose up --build -d`

---

Генерация случайных данных в БД:

`docker compose exec api python manage.py generate_data`

---

Регистрация суперпользователя:

`docker compose exec api python manage.py createsuperuser`

---

Swagger для тестов эндпоинтов;
http://localhost:8000/v1/docs/
