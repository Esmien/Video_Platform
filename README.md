# **Проект реализует API видео платформы, на которой пользователи могут публиковать свои видео**

# **Функционал**:

### Генерация тестовых данных

Скрипт `generate_data` генерирует 10.000 пользователей, 100.000 видео и около 300.000 распределенных лайков с защитой от
дубликатов.

<u>Есть небольшая вероятность самолайка</u> в сгенерированных данных, от защиты и дополнительных проверок осознанно <u>
отказался ради скорости</u> выполнения скрипта.

### Работает по принципу:

#### 1. Пользователи

> 1. Создание списка из 10.000 объектов пользователей с уникальными username типа `user_N` в памяти Python
>2. Вставка в таблицу `auth_user` пачками по 2.000 записей за раз
>3. Получение списка `id (int)` всех сгенерированных пользователей в список

#### 2. Видео

> Та же логика у генерации видео, кроме автора (`owner`) -
> внешний ключ `owner_id` назначается вручную случайным образом из списка `id` всех сгенерированных пользователей, пачки
> по 5.000

#### 3. Лайки

> Случайным образом выбирается пара `user_id`, `video_id`, выполняется проверка, не было ли такой пары ранее (при помощи
> множества),
> если не было, то создается объект лайка с полями `user_id`, `video_id`, в которые записываются значения из отобранной
> ранее пары.
> После этого так же, пачками уже по 10.000 (для скорости скрипта, БД справляется), записываются в БД

#### 4. Синхронизация

> В финале выполняется подсчет и простановка количества лайков в поле `total_likes` у каждого видео.
>

>**Логика:**
> 
>В подзапросе через `annotate` и `Count` считается количество лайков (по их `id`) для каждого конкретного видео (его `id` прокидывается из внешнего запроса при помощи `OuterRef`).
Затем в основном запросе это количество массово присваивается полю `total_likes` с аккуратной обработкой `null` через `Coalesce` (если у видео вообще нет лайков, подставляется 0).

# **Админка**

### Работа с записями о видео:

1. #### **Отображаемые поля:**

* `id`
* название(`name`)
* автор(`owner`)
* статус(`is_published`)
* всего лайков(`total_likes`)
* дата создания(`created_at`)

2. #### **Фильтрация:**

* по статусу(`is_published`)
* по дате создания(`created_at`)

3. #### **Поля для поиска:**

* название(`name`)
* никнейм автора(`user_username`)

4. #### **Неизменяемые поля:**

* дата создания(`created_at`)
* всего лайков(`total_likes`)

###### Авторов подтягивает через select_related, чтобы избежать повторного обращения к таблице auth_user при получении поля 'owner' и имени из него на каждом видео

---

### Работа с файлами видео:

1. #### **Отображаемые поля:**

* `id`
* ссылка на связываемое с файлом видео(`video`)
* качество(`quality`)
* ссылка на файл(`file`)

2. #### **Фильтрация:**

* по качеству(`quality`)

3. #### **Поля для поиска:**

* название видео(`video_name`)

###### Записи связанных описаний подтягивает через select_related, чтобы не бегать каждый раз в таблицу api_video для получения описания связанного видео

---

### Работа с лайками:

1. #### **Отображаемые поля:**

* `id`
* видео, на котором стоит лайк(`video`)
* кто этот лайк поставил(`user`)

2. #### **Поля для поиска:**

* название лайкнутого видео(`video_name`)
* никнейм лайкнувшего пользователя(`user_username`)

###### Записи связанных видео и пользователей подтягиваются через select_related

# **Как запустить**

---
#### Перейти в корень проекта (video_platform)

```bash 
cd ./video_platform
```

---

### Настройка окружения

#### Создать файл с переменными окружения:

```bash 
cp .env.example .env
```

#### Сгенерировать секретный ключ, скопировать и вставить в .env в переменную **SECRET_KEY**:

```bash
openssl rand -base64 50
```

При необходимости изменить базовые секреты БД на свои

---

### Запуск проекта

```bash
docker compose up --build -d
```

---

### Генерация случайных данных в БД

```bash
docker compose exec api python manage.py generate_data
```

---

### Регистрация суперпользователя

```bash
docker compose exec api python manage.py createsuperuser
```

---
