# **Проект реализует API видео платформы, на которой пользователи могут публиковать свои видео**

**Функционал**:
=

* ### Регистрация

POST `/v1/register/`

---

* ### Добавление видео 
(через админку http://localhost:8000/admin)

---

* ### Получения списка с деталями опубликованных видео, а также неопубликованных, если запрашивающий - их автор 
 
GET `/v1/videos/`

---

* ### Получение деталей о видео по его ID, если оно опубликовано или запрашивается автором. Работает query-запрос ?user_expand=true

GET `/v1/videos/{id}`

GET `/v1/videos/{id}?user_expand=true`

---

* ### Постановка лайка с защитой от дубликатов, самолайков и установки на неопубликованное видео

POST `/v1/videos/{id}/likes/`

---

* Получение списка ID всех опубликованных видео

GET `/v1/videos/ids/`

---

* Получение статистики (видео-лайки) методами подзапроса и GROUP BY 

GET `/v1/videos/statistics-subquery/` и 

GET `/v1/videos/statistics-group-by/`

---

# **Админка**

### Работа с записями о видео:

1. #### **Отображаемые поля:**

* id
* название(name)
* автор(owner)
* статус(is_published)
* всего лайков(total_likes)
* дата создания(created_at)

2. #### **Фильтрация:**

* по статусу(is_published)
* по дате создания(created_at)

3. #### **Поля для поиска:**
* название(name)
* никнейм автора(user_username)

4. #### **Неизменяемые поля:**
* дата создания(created_at)
* всего лайков(total_likes)

###### Авторов подтягивает через select_related, чтобы избежать повторного обращения к таблице auth_user при получении поля 'owner' и имени из него на каждом видео

---

### Работа с файлами видео:
1. #### **Отображаемые поля:** 
* id
* ссылка на связываемое с файлом видео(video)
* качество(quality)
* ссылка на файл(file)
2. #### **Фильтрация:**
* по качеству(quality)
3. #### **Поля для поиска:**
* название видео(video_name)

###### Записи связанных описаний подтягивает через select_related, чтобы не бегать каждый раз в таблицу api_video для получения описания связанного видео

---

### Работа с лайками:
1. #### **Отображаемые поля:** 
* id
* видео, на котором стоит лайк(video)
* кто этот лайк поставил(user)
2. #### **Поля для поиска:**
* название лайкнутого видео(video_name)
* никнейм лайкнувшего пользователя(user_username)
###### Записи связанных видео и пользователей подтягиваются через select_related

---

# **Как запустить**

---
Перейти в корень проекта (video_platform)

`cd ./video_platform`

---

Настройка окружения:

`cp .env.example .env` - создать файл с переменными окружения

`openssl rand -base64 50` - сгенерировать секретный ключ, скопировать и вставить в .env в переменную **SECRET_KEY**

При необходимости изменить базовые секреты БД на свои

---

Запуск проекта:

`docker compose up --build -d`

---

Генерация случайных данных в БД:

`docker compose exec api python manage.py generate_data`

---

Регистрация суперпользователя:

`docker compose exec api python manage.py createsuperuser`

---

Swagger для тестов эндпоинтов;
http://localhost:8000/v1/docs/
